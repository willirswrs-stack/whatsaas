# Work Log - 19 de Janeiro de 2026

## Sessão: Implementação de Mídia, IA e Duplicação de Campanhas

### ✅ Funcionalidades Implementadas

#### 1. Sistema de Upload de Mídia
- **Novo módulo `UploadsModule`** em `backend/src/modules/uploads/`
  - `uploads.controller.ts` - Endpoint `POST /api/v1/uploads/media`
  - `uploads.service.ts` - Salva arquivos no disco e gera URLs acessíveis
- **Configuração de arquivos estáticos** em `main.ts` para servir `/uploads`
- **Tipos de arquivo suportados:**
  - Vídeo: MP4, WebM, MOV
  - Imagem: JPEG, PNG, GIF, WebP
  - Áudio: MP3, WAV, OGG, MP4
  - Documentos: PDF, DOC, DOCX, XLS, XLSX, PPT, PPTX
- **Limite de tamanho:** 60MB

#### 2. Envio de Mídia via WhatsApp
- **Método `sendMedia`** adicionado à interface `IWhatsAppProvider`
- **Implementação no `EvolutionAdapter`** - Usa endpoint `/message/sendMedia`
- **Implementação no `WahaAdapter`** - Usa endpoint `/api/default/sendFile`
- **Suporte no `FlowsService.processExecution`** para nós de mídia (video, image, audio, document)

#### 3. Frontend - Upload Automático
- **Componente `MediaUploadArea`** atualizado para:
  - Fazer upload automático para o backend ao selecionar arquivo
  - Mostrar spinner durante upload
  - Exibir erros de upload
  - Retornar URL do arquivo para o nó do fluxo

#### 4. Duplicação de Campanhas
- **Backend:**
  - Endpoint `POST /api/v1/campaigns/:id/duplicate`
  - Método `duplicate()` no `CampaignsService`
  - Copia campanha, contatos e reseta status para 'draft'
- **Frontend:**
  - Método `duplicate()` no `campaignsService`
  - Botão de duplicar (ícone azul) na listagem de campanhas
  - Feedback de sucesso ao duplicar

#### 5. Fallback de API Key para IA
- **FlowsService** agora usa `OPENAI_API_KEY` do `.env` como fallback
- Se o nó de IA não tiver token configurado, usa a chave do ambiente
- Adicionado `ConfigService` ao `FlowsService`

### 📝 Notas

- O backend precisa estar rodando em `0.0.0.0:3333` para o Docker acessar
- Os arquivos de mídia são salvos em `backend/uploads/`
- A Evolution API acessa os arquivos via `http://192.168.0.8:3333/uploads/`
- Se mudar de rede, atualizar `PUBLIC_HOST` no `.env` e reiniciar backend
