# Relatório de Trabalho - 22/01/2026

## ✅ Resumo
Foco total na estabilização do **Editor de Fluxos (Flow Builder)** e na **Lógica de Execução (Backend)**. Foram corrigidos bugs que impediam a deleção de conexões, salvamento de nós específicos e execução de disparos.

## 🛠️ Alterações Realizadas

### 1. Frontend: Editor de Fluxos
**Arquivo:** `frontend/src/app/flows/[id]/page.tsx`

*   **Infraestrutura ReactFlow**:
    *   Adicionado wrapper `<ReactFlowProvider>` no componente da página. Isso corrigiu os erros onde hooks como `useReactFlow` falhavam dentro de sub-componentes.
    *   Refatoração da exportação do componente principal.

*   **Gerenciamento de Arestas (Edges)**:
    *   Reescrito o componente `DeletableEdge`.
    *   **Correção**: Substituição de HTML (`foreignObject` + `button`) por SVG nativo (`<g>`, `<circle>`, `<path>`). Isso resolveu o problema onde o botão de deletar conexão não era clicável ou desaparecia.

*   **Nós de SMS e Email**:
    *   **Problema**: Estes nós existiam visualmente mas não tinham lógica de dados.
    *   **Solução**:
        *   Integrado hook `useNodeConfig` persistir dados.
        *   Adicionado `useEffect` com *debounce* para salvar alterações automaticamente.
        *   Implementadas as funções dos botões **Duplicar** e **Remover** que antes eram apenas visuais.

*   **UI/UX**:
    *   Melhoria nos "handles" de redimensionamento das sidebars (esquerda e direita).
    *   Aumento da área de clique (4px -> 12px) e adição de indicador visual para facilitar o uso.

### 2. Backend: Execução e Limites
**Arquivo:** `backend/src/modules/flows/flows.service.ts`

*   **Processamento de Nós**:
    *   Adicionado suporte no método `processExecution` para nós dos tipos `sms` e `email`.
    *   **SMS**: Implementado como envio de mensagem WhatsApp com prefixo `[SMS]` (para fins de simulação/teste) usando o provider conectado.
    *   **Email**: Implementado log de simulação de envio (preparado para futura integração SMTP).
    *   Melhoria no log de erro para nós desconhecidos, agora exibindo o tipo real do nó (`data.type`) facilitando o debug.

**Arquivo:** `backend/src/scripts/reset-daily-limits.ts` (Novo)

*   **Gestão de Instâncias**:
    *   Criado script utilitário para desbloquear instâncias que atingiram o limite diário de mensagens.
    *   Reseta `daily_sent` para 0 e define `daily_limit` para 1000.
