## Problemas Resolvidos

### 1. Falha no Envio de Mensagens via Fluxo
- **Sintoma:** Campanhas rodavam, mas mensagens não chegavam. Status ficava preso ou falhava silenciosamente.
- **Causa Raiz:** O serviço `FlowsService` criava o registro de execução mas não tinha a lógica implementada para processar os nós de mensagem e realizar o envio efetivo.
- **Solução:** Implementada lógica síncrona no `FlowsService.processExecution` para processar o nó inicial, buscar a instância e contato, e disparar a mensagem usando o `WhatsAppProvider`.

### 2. Erro de Instância Incorreta (WWebJS vs Evolution)
- **Sintoma:** Logs mostravam erro `WWebJS API error` mesmo em campanhas configuradas para `Evolution API`.
- **Causa:** Registros antigos na tabela `campaign_contacts` mantinham vínculo com instâncias antigas (WWebJS) e sobrescreviam a configuração da campanha.
- **Solução:**
  - Realizada limpeza no banco de dados (`UPDATE campaign_contacts SET instance_id = NULL...`) para forçar o uso da instância correta da campanha.
  - Resetados os status dos contatos falhos para `queued` para permitir reenvio.

### 3. Erro na Criação de Campanha (Bad Request)
- **Causa:** Payload do frontend enviava `minDelay` dentro de `settings`, mas o backend esperava na raiz.
- **Solução:** Ajustado `page.tsx` e DTOs para enviar os campos corretamente.
