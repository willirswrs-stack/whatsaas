## ✅ Módulos Implementados

### 1. 🛡️ Anti-Ban Module (`/src/modules/anti-ban/`)

Novo módulo criado com 4 serviços essenciais:

#### 1.1 `HumanBehaviorService` (HBS Engine)
- **Função:** Simula comportamento humano no envio de mensagens
- **Features:**
  - Cálculo de duração de digitação baseado em WPM (20-60 palavras/min)
  - Pausas naturais entre frases
  - Verificação de janela horária (ex: 08:00-20:00)
  - Callbacks para simular status "digitando" no WhatsApp

#### 1.2 `DelayGeneratorService`
- **Função:** Gera delays realísticos entre mensagens
- **Features:**
  - Distribuição Gaussiana (normal) para delays
  - Distribuição Exponencial (opcional)
  - Jitter de ±15% para evitar padrões detectáveis
  - Cálculo de delay adaptativo para warmup (chips novos)
  - "Segundos quebrados" (ex: 47.832s em vez de 48s)

#### 1.3 `PatternBreakerService`
- **Função:** Quebra padrões detectáveis pelas defesas do WhatsApp/Meta
- **Features:**
  - Pool de saudações variadas (formal, casual, direto)
  - Randomização de emojis
  - Variação de pontuação
  - Hash SHA256 do conteúdo para anti-duplicata
  - Detector de mensagens repetidas

#### 1.4 `PhoneNormalizerService`
- **Função:** Normaliza telefones brasileiros para formato E.164
- **Features:**
  - Reconhece DDDs brasileiros (55+ estados)
  - Adiciona +55 automaticamente
  - Diferencia celular de fixo
  - Remove duplicatas
  - Suporte a import de CSV/TXT
  - Estatísticas de importação

#### 1.5 `StackRouterService` ⭐ **NOVO**
- **Função:** Motor de decisão inteligente que seleciona qual stack WhatsApp usar
- **Princípio:** "Toda inteligência vive no WhatSaas, não nas APIs"
- **Stacks Disponíveis:**
  - `wwebjs` - Alto mimetismo humano (browser real)
  - `waha` - API HTTP sobre WhatsApp Web (balanceado)
  - `evolution` - Headless, alta escala
  - `official` - API Oficial Meta (institucional)
- **Inputs Avaliados:**
  - `warmupDays`, `chipHealthScore`, `daysSinceLastBlock`
  - `campaignVolume`, `riskLevel`, `conversationActive`
- **Sistema de Regras:**
  - 15+ regras priorizadas por criticidade
  - Confidence scoring (0-100%)
  - Fallback stack automático
- **Features:**
  - Batch routing para campanhas
  - Scenario recommendations
  - Stack capabilities comparison
- **Testes:** 23/23 passando ✅

#### 1.6 `AntiBanAnalyticsService` ⭐ **NOVO**
- **Função:** Motor de observabilidade para métricas anti-ban em tempo real
- **Eventos Rastreados:**
  - `sent`, `delivered`, `read`, `failed`, `blocked`
- **Métricas por Instância:**
  - `totalSent`, `deliveryRate`, `failureRate`, `blockRate`
  - `healthScore` (0-100) com trends
  - `averageDeliveryTimeMs`
- **Métricas por Stack:**
  - Taxa de sucesso por stack (waha, wwebjs, evolution)
  - Volume por hora
  - Latência média
- **Detecção de Anomalias:**
  - Spike de falhas (>10% warning, >25% critical)
  - Detecção de bloqueio
  - Queda de delivery rate
- **Dashboard:**
  - Overview 24h
  - Volume por hora
  - Top instances
  - Health trends com previsão
- **Export Prometheus:**
  - Métricas compatíveis com Grafana
  - `whatsaas_instance_sent_total`, `whatsaas_stack_success_rate`
- **Testes:** 21/21 passando ✅

#### 1.7 `AnalyticsController` (API REST)
- **Endpoints:**
  - `GET /analytics/dashboard` - Dashboard completo
  - `GET /analytics/instances` - Métricas de instâncias
  - `GET /analytics/stacks` - Performance por stack
  - `GET /analytics/alerts` - Alertas e anomalias
  - `GET /analytics/metrics/prometheus` - Export Prometheus
  - `GET /analytics/router/recommend` - Recomendação de stack

#### 1.8 `ChipHealthService` ⭐ **NOVO**
- **Função:** Calculadora sofisticada de saúde do chip (0-100)
- **Fatores Ponderados:**
  - `Delivery Success` (35%): Taxa de entrega vs meta
  - `Warmup Maturity` (25%): Progresso do aquecimento (0-21 dias)
  - `Block History` (20%): Histórico de banimento/desconexão
  - `Usage Capacity` (10%): Uso vs Limite diário (ideal 40-60%)
  - `Activity Recency` (10%): Tempo desde última atividade
- **Status de Saúde:**
  - 🟢 `Excellent` (85-100): Capacidade total (+20% limite)
  - 🟡 `Good` (70-84): Operação normal
  - 🟠 `Moderate` (50-69): Reduzir volume (-20% limite)
  - 🔴 `Poor` (30-49): Risco alto (-50% limite)
  - ⛔ `Critical` (0-29): Pausar envios
- **Features:**
  - Recomendações automáticas baseadas nos fatores baixos
  - Cache de score para performance
  - Visão geral por Tenant

#### 1.9 `WarmupEngine` (Service + Processor) ⭐ **NOVO**
- **Função:** Gerencia o ciclo de vida e aquecimento dos chips
- **Mecanismo:**
  - `WarmupService`: Define a curva de maturação (14 dias)
  - `WarmupProcessor`: Executa tarefas agendadas via BullMQ
  - `Scheduler`: Job diário (04:00 AM) para avançar estágios
- **Curva de Warmup:**
  - Dia 1: 10 msg (intervalo 120s)
  - Dia 7: 160 msg (intervalo 20s)
  - Dia 14: 1200 msg (Graduação 🎓)
- **Features:**
  - Avanço automático de dias e limites
  - Proteção contra avanço se desconectado
  - Graduação automática para chips maduros (Warmup Disabled)

  - Graduação automática para chips maduros (Warmup Disabled)
